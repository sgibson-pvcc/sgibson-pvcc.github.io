Operating Systems Terminology

TERM DEFINITION
Operating Systems the software that controls the overall operation of a computer
Job queue a storage organization in which objects (in this case, jobs) are ordered
FIFO the objects are removed from the queue in the order in which they arrived
Interactive processing new operating systems were developed that allowed a program being executed to carry on a dialogue with the user through remote terminals
time-sharing provides service to multiple users at the same time
multiprogramming time is divided into intervals and then the execution of each job is restricted to only one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval
application software consists of the programs for performing tasks particular to the machine’s
system software provides the infrastructure that the application software requires
ultility software consists of programs for performing activities that are fundamental to computer installations but not included in the operating system. In a sense, utility software consists of software units that extend (or perhaps customize) the capabilities of the operating system
graphical User interface (GUI) objects to be manipulated, such as files and programs, are represented pictorially on the display as icons. These systems allow users to issue commands by using one of several common input devices. For example, a computer mouse, with one or more buttons, can be used to click or drag icons on the screen. In place of a mouse, special-purpose pointing devices or styluses are often used by graphic artists or on several types of hand-held devices.>
shell communicated with users through textual messages using a keyboard and monitor screen
kernel contains those software components that perform the very basic functions required by the computer installation
device driver software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine. Each device driver is uniquely designed for its particular type of device (such as a printer, disk drive, or monitor) and translates generic requests into the more technical steps required by the device assigned to that driver.
file manager the software that controls the overall operation of a computer
memory manager the task of coordinating the machine’s use of main memory
vertual memory where the computer is able to function as though it actually had 8GB of main memory.
boot strapping procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory (which is essentially empty when the machine is first turned on)
ROM a small portion of a computer’s main memory where the CPU expects to find its initial program is constructed from special nonvolatile memory cells.
deadlock the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling where the operating system would make the nonshareable resource appear shareable by creating the illusion of more than one printer.
super user (or administrator) person who gains highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure
multi-user athentication One solution to the pervasive problem of humans using weak password to secure their computer accounts is to require more than one piece of evidence to gain access.
